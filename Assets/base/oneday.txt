# 어느 날 갑자기 - 프로젝트 개발 로그 (Development Log)

## 🚀 구현된 핵심 기능 (Core Features)
- **림버스 컴퍼니 스타일 던전 탐사**: DAG 기반 맵 생성, 노드 간 연결성 검증 및 단방향 진행 시스템 구축.
- **거점(Base) 시스템**: 거점 건설 및 방어 로직, 거점 반경에 따른 탐사 가능 범위 제한 시스템.
- **이벤트 및 보상 체계**: 12종 이상의 다양한 이벤트 시나리오, 난이도별 보상 스케일링, 태그 기반([HPDamage:10] 등) 결과 파싱.
- **전투 및 동료 시스템**: 동료 전투력을 기반으로 한 자동 전투 결과 계산, 패배 시 자원 손실 및 강제 후퇴 패널티.
- **안개(Fog of War) 및 시각화**: 가시성 제어, 방문 노드 표시, 노드 타입별 컬러 코딩 및 UI 레이아웃 안정화.
- **코드 모듈화**: 비대해진 ExploreManager를 EventScenarioBuilder 등으로 분리하여 유지보수성 확보.

- **겪었던 어려움**: 
  1. 리팩토링 도중 `PanToNode` 등 핵심 카메라 제어 메서드가 유실되어 `CS0103` 컴파일 에러 발생.
  2. 노드 진입 시 표시되는 이벤트 UI에서 텍스트가 줄바꿈 범위를 벗어나거나 버튼과 겹쳐서 클릭할 수 없는 UI 붕괴 현상 발생.
  3. 맵 진입 시 카메라가 항상 시작 노드에서 우측으로 심하게 치우치는 버그.
- **어떻게 해결했나**: 
  1. 누락된 메서드를 신속히 재작성하고 참조를 복구하여 컴파일 에러 해결.
  2. UI 요소들의 Y좌표를 강제로 할당 및 고정시키는 레이아웃 스크립트를 적용하여 이벤트 창의 가독성을 확보하고 겹침 문제 해결.
  3. 카메라 초기화 로직을 추적하여 의도치 않게 들어가 있던 오프셋 값(+400f)을 찾아내 제거함으로써 시작 노드 중앙 정렬 완벽 구현.

### 2026-04-15: 코루틴 충돌 및 터치 인터페이스 최적화
- **무엇이 바뀌었나**: 노드 타입별 컬러 코딩, 안개(Fog of War) 시스템 추가로 탐사 시각 효과 강화.
- **겪었던 어려움**: 
  1. 화면이 어두워지는 `FadeDim` 효과가 중복 실행되면서 연출이 깨지거나 멈추는 코루틴 충돌 문제.
  2. 이동 애니메이션 도중 유저가 노드를 마구 클릭하면 이벤트가 2~3번 중복 트리거되는 치명적인 상태 관리 오류.
  3. 맵의 노드들을 잇는 연결선(LineRenderer/UI Line)이 마우스 클릭 레이캐스트를 가로채어 노드가 클릭되지 않는 조작감 문제.
- **어떻게 해결했나**: 
  1. 기존의 문자열 기반 코루틴 제어 방식을 안전한 변수 추적 기반(`Coroutine` 객체 저장)으로 변경하여 중복 실행을 막고 안전하게 중단/재시작되도록 수정.
  2. `isMovingToNode`라는 상태 가드 플래그를 도입하여, 플레이어가 이동 중일 때는 클릭 입력을 완전히 무시하도록 원천 차단.
  3. 연결선의 UI 컴포넌트에서 `raycastTarget` 옵션을 비활성화하여, 클릭 판정이 오직 노드 UI에만 도달하도록 최적화.

### 2026-04-20: 무한 탐험 제한 및 거점(Base) 시스템 도입
- **무엇이 바뀌었나**: 탐험 가능 범위를 제한하는 '거점 시스템'의 기초 구현. 거점 건설, A-site 기반 순차적 반경 확장, 거점 파괴 로직 추가.
- **겪었던 어려움**: 초기 설계에서는 유저가 제약 없이 끝없이 맵을 탐사할 수 있어 리소스(자원) 경제와 생존의 긴장감이 완전히 무너짐.
- **어떻게 해결했나**: 노드와 상호작용하기 위한 선행 조건으로 '인근에 활성화된 거점이 있는가?'를 검사하는 로직을 `ExploreManager`에 추가. 자원을 소모해 거점을 지어야만 탐사 반경이 넓어지도록 강제하여 게임의 핵심 탐색 루프(파밍 -> 거점 건설 -> 확장)를 확립.

### 2026-04-21: 자동 전투 패널티 및 자원 복사 버그 수정
- **무엇이 바뀌었나**: B, M, K 구역 등 설정에 맞춘 지구별 난이도 배치, 동료 전투력 기반 자동 전투 연출(2초 딜레이 추가), 전투 패배 로직(자원 20% 손실 및 거점 후퇴), 기록/포션 아카이브 기반 추가.
- **겪었던 어려움**: 
  1. UI 패널을 빠르게 닫거나 특정 조건에서 상호작용 시 자원(Food, Material)이 중복으로 지급되는 복사 버그 발견. 이로 인해 2-거점 체제 진행 밸런스가 파괴됨.
  2. 전투 결과가 즉각적으로 뜨다 보니 플레이어가 상황을 인지하지 못하고, 패배 시에도 아무런 타격이 없어 긴장감이 부족.
- **어떻게 해결했나**: 
  1. `ExploreManager`와 `BaseManager` 간의 보상 지급 시점과 상태 변화 동기화 로직을 대대적으로 감사(Audit). 패널 닫기 시 이벤트 처리가 완료되었는지 검증하는 플래그를 추가하고, 초기 보상량을 대폭 삭감해 경제 밸런스를 맞춤.
  2. 2초간의 전투 연출 딜레이를 주어 몰입감을 높이고, 패배 시 근처 거점으로 강제 텔레포트 및 보유 자원 20% 증발 패널티를 구현하여 전투의 무게감을 부여.

### 2026-04-22: 안전 모드(Safe Mode) 에러 복구 및 거대 클래스 리팩토링
- **무엇이 바뀌었나**: `EventScenarioBuilder.cs` 분리 생성, 태그 기반 파싱 로직(`[HPDamage]`, `[Recruit]` 등) 도입, 맵 상태 계산 로직 최적화.
- **겪었던 어려움**: 
  1. 프로젝트를 열었을 때 `GameManager.cs`의 참조 누락 및 `AllyData.cs`의 클래스 중복 선언 문제로 유니티가 안전 모드(Safe Mode)로 강제 진입하며 아무 작업도 할 수 없는 상태 발생.
  2. `ExploreManager.cs`가 길고 복잡한 이벤트 텍스트와 분기 로직을 모두 끌어안은 수천 줄 짜리 '신(God) 클래스'가 되어 사소한 수정에도 버그가 연쇄 발생함.
  3. 기존 텍스트 기반 보상 파싱 로직이 불안정하여 오타 하나에 시스템이 뻗어버림.
- **어떻게 해결했나**: 
  1. 문제가 된 파일들의 네임스페이스와 클래스 선언부를 즉각 수정하여 유니티 컴파일 오류를 해소하고 안전 모드에서 탈출함.
  2. 방대한 텍스트 생성 및 시나리오 분기 로직을 통째로 뜯어내어 `EventScenarioBuilder.cs`라는 모듈로 독립시킴. 이로 인해 `ExploreManager`의 코드가 300줄 이상 경량화되며 가독성이 살아남.
  3. 오작동이 잦던 문자열 검색 대신 `[HPRecover:20]`와 같은 대괄호 기반 직관적 태그 파서를 구현하여, 디버깅이 쉽고 기획 데이터를 넣기 편한 구조로 확립함.

### 2026-05-06: 자원 획득 밸런스 전면 조정 및 패널티 버그 수정
- **무엇이 바뀌었나**: `EventScenarioBuilder.cs` 내의 보상 계산 로직 리팩토링, 전체적인 자원(Material, Food, Record) 획득량 상향, 전투 패배 시 자원 손실 패널티 계산 버그 수정.
- **겪었던 어려움**: 
  1. 하드코딩된 보상 수치들 때문에 관리가 힘들었고, 탐험 대비 자원 획득량이 적어 거점 건설(50 Material 요구)과 생존이 팍팍하여 밸런스 불균형이 있었음.
  2. 전투 패배 시 식량과 자원을 각각 20%씩 잃어야 하는데, 코드 작성 오류로 인해 식량 손실량을 자원(Material)에서 가져오고, 자원 손실량을 식량(Food)에서 가져오는 치명적인 논리 버그가 존재했음.
- **어떻게 해결했나**: 
  1. 분산되어 있던 보상 연산(`Mathf.RoundToInt(x * mult)`)을 `CalcReward()`라는 단일 함수로 추상화하여 코드 가독성과 유지보수성을 크게 높임.
  2. 전투 보상 및 주요 파밍 이벤트(상자, 트럭 등)의 기본 보상값을 상향 조정하여 초보자도 부드럽게 기반을 다질 수 있도록 경제 밸런스를 개편함.
  3. `ExploreManager.cs`의 `ApplyDefeatPenalty()` 함수 내부에서 `lostFood`와 `lostMat` 산출 변수가 서로 바뀌어 있던 코드를 올바르게 수정하여, 정확히 자신의 자원에서 20%가 차감되도록 바로잡음.

### 2026-05-19: 거점 기습 텔레포트 버그 수정 및 보스방 조우 시점 조정
- **무엇이 바뀌었나**: 이벤트 선택지 중복 실행에 따른 강제 귀환 버그 해결, 거점 기습 이벤트(`CheckBaseAttack`)가 보스방(마지막 레이어) 전에는 등장하지 않도록 제한. 유니티 콘솔에 쌓이는 Missing Script 오류 정리용 에디터 툴 도입.
- **겪었던 어려움**: 
  1. 기지를 짓고 다음 노드로 진행하려 할 때, 플레이어가 갑자기 기지로 강제 텔레포트되는 심각한 이동 동기화 버그가 발생함.
  2. 에디터 로그 상에 지속적으로 'The referenced script on this Behaviour is missing!' 경고가 쌓여 다른 오류를 파악하기 어려움.
- **어떻게 해결했나**: 
  1. 유니티 UI 버튼을 템플릿으로 복제하는 과정에서 과거 설정해 둔 리스너(`EnterNode()`)가 이벤트 처리 함수(`OnChoiceSelected()`)와 중복 실행되는 것을 발견. 이벤트 노드일 때는 즉시 이동 로직이 무시되도록 가드(Guard)를 추가함.
  2. 기습 이벤트가 플레이어의 목표 노드(`selectedNode`) 변수를 기지 위치로 덮어씌워버리는 문제를 파악하고 해당 로직을 분리.
  3. 기습 방어 이벤트의 등장을 보스방 이전에는 아예 차단하여 플레이어의 초중반 탐사 피로도를 대폭 낮춤.
  4. `Assets/Editor/RemoveMissingScripts.cs` 스크립트를 추가하여 상단 메뉴의 [Tools -> Remove Missing Scripts]를 누르면 씬 내의 찌꺼기 스크립트들이 일괄 청소되도록 개선함.

### 2026-06-02: 탐사 씬 전면 리팩토링 및 비주얼 폴리싱 완료
- **무엇이 바뀌었나**:
  1. `ExploreManager.cs`를 1333줄에서 약 500줄로 대폭 경량화. 맵 생성 및 선 그리기 로직을 `ExploreMapGenerator.cs`로 완전 분리하여 역할별 단일 책임 원칙(SRP) 적용.
  2. 노드 간 연결선(DrawLines) 수학 공식을 완전히 재작성. 기존 `pivot=(0, 0.5)` + 시작점 고정 방식에서 두 노드의 **중점(Midpoint)**에 배치하는 `pivot=(0.5, 0.5)` 방식으로 교체하여, 어떤 방향의 연결선도 두 노드를 정확히 잇도록 수학 오류 완전 수정.
  3. 노드 아이콘(IconCombat, IconEvent, IconBase) 및 플레이어 이동 아이콘(IconPlayer), 맵 배경(ExploreBG)을 `Resources.Load`로 코드에서 강제 로드하는 방식에서, `ExploreManager` Inspector에 공개 `Sprite` 변수를 열어 **드래그앤드롭으로 직접 할당**하는 방식으로 전환. (Unity 권장 방식)
  4. `MapDrag.cs`에 드래그 거리 임계값(8px)을 도입. 기존에는 마우스 버튼을 누르는 순간 드래그로 처리되어, 노드를 가볍게 클릭해도 드래그로 오인하는 조작감 문제가 있었음. 임계값 초과 시에만 드래그로 확정하여 **노드 클릭과 맵 드래그 충돌 완전 해소**.
  5. `Node.cs`에서 드래그 중 클릭 이벤트가 발생하면 `MapDrag.IsDragging` 플래그를 확인하여 클릭을 무시하도록 방어 로직 추가.
  6. 탐사 맵 배경 이미지 및 디스토피아 스타일 노드 아이콘 5종(ExploreBG, IconCombat, IconEvent, IconBase, IconPlayer)을 AI 생성 이미지로 제작하여 `Assets/Resources/Sprites/Nodes/` 폴더에 배치.
